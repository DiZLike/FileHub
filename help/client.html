Документация FileHub Client

📌 Обзор клиента

FileHub Client — это консольное приложение для взаимодействия с сервером FileHub. Клиент позволяет пользователям подключаться к серверу, создавать раздачи (файлы и папки), просматривать доступные раздачи, скачивать файлы других пользователей и управлять своими раздачами.

Ключевые возможности:

  • 🔐 Подключение с аутентификацией и сохранением пароля
  • 📤 Расшаривание файлов и папок с автоматическим сканированием содержимого
  • 📋 Просмотр всех доступных раздач с информацией о владельце
  • 📥 Скачивание файлов с отображением прогресса
  • 🔄 Автоматическая синхронизация локальных раздач с сервером
  • 💾 Локальное сохранение информации о раздачах в JSON-файлах
  • 📊 Просмотр статистики сервера
  • 🎨 Удобный консольный интерфейс с разделением зон вывода

🏗️ Архитектура клиента

Клиент построен по модульному принципу с четким разделением ответственности:

🧠 client.py

Главный класс, инициализация компонентов, основной цикл.

🔌 connection.py

Сетевое взаимодействие, отправка/прием JSON, канал данных.

📤 shares.py

Управление раздачами, синхронизация, локальное хранение.

📥 downloads.py

Скачивание файлов и папок с сервера.

📡 uploads.py

Отправка файлов по запросу других пользователей.

🔑 auth.py

Аутентификация, сохранение хешей паролей.

⚙️ config.py

Загрузка и управление конфигурацией клиента.

🖥️ ui.py

Пользовательский интерфейс, меню, обработка ввода.

🎨 console_ui.py

Управление консольным выводом, прогресс-бары.

📡 protocols.py

Константы протокола обмена с сервером.

🛠️ utils.py

Вспомогательные функции форматирования и работы с файлами.

🚀 Быстрый старт

# Запуск клиента
python client.py

При первом запуске будет создан файл конфигурации client.conf. После ввода имени пользователя и пароля клиент подключается к серверу и отображает главное меню.

Основные команды меню:

  1. Поделиться файлом/папкой — указать путь к локальному файлу или папке
  2. Список всех раздач — просмотр доступных раздач на сервере
  3. Мои раздачи — просмотр своих раздач с локальными путями
  4. Скачать раздачу — скачать файл по ID раздачи
  5. Удалить раздачу — удалить свою раздачу с сервера
  6. Статистика сервера — информация о работе сервера

📦 Детальное описание модулей

client.py Главный модуль

Класс FileHubClient инициализирует все компоненты и предоставляет методы для основных действий: подключение, отключение, создание раздачи, скачивание. Запускает фоновый поток для периодического пинга сервера с ID активных раздач.

connection.py Сетевое подключение

ConnectionManager управляет TCP-соединением с сервером. Отправляет JSON-сообщения с префиксом длины (4 байта). Поддерживает синхронные команды с ожиданием ответа через _response_event и асинхронные уведомления (запросы на отправку файла) через колбэк on_upload_request.

shares.py Управление раздачами

ShareManager отвечает за создание, хранение и синхронизацию раздач. Локальные раздачи сохраняются в JSON-файлах в подпапке пользователя (my_shares/{username}/). Поддерживает миграцию старых файлов из корневой папки. При подключении может автоматически синхронизировать локальные раздачи с сервером.

downloads.py Скачивание файлов

DownloadManager обрабатывает запросы на скачивание. Подключается к каналу данных сервера, принимает файлы и сохраняет их в директорию загрузок. Поддерживает отображение прогресса скачивания.

uploads.py Отправка файлов

UploadManager обрабатывает входящие запросы на отправку файлов от других пользователей. Подключается к каналу данных как отправитель и передает файлы получателю.

auth.py Аутентификация

AuthManager управляет сохранением хешей паролей в файле client_pass.hash. Позволяет запоминать пароль для последующих подключений.

config.py Конфигурация

ClientConfig загружает параметры из client.conf. Предоставляет свойства для доступа к настройкам подключения, загрузок, интерфейса и безопасности.

ui.py Пользовательский интерфейс

UserInterface реализует консольное меню с разделением зон вывода. Поддерживает очередь уведомлений и фоновый поток для их отображения.

console_ui.py Управление консолью

ConsoleManager предоставляет методы для очистки экрана, вывода прогресс-баров, форматирования сообщений. Глобальный экземпляр console используется всеми модулями.

protocols.py Протоколы

Определяет классы MessageActions, ShareTypes, StatusCodes и константы размеров буферов.

utils.py Утилиты

Статические методы для форматирования размера, хеширования паролей, рекурсивного сканирования папок и безопасного удаления файлов.

🧩 Расширение клиента: добавление новых модулей

Модульная архитектура клиента позволяет легко добавлять новую функциональность. Рассмотрим пример добавления модуля для работы с избранными раздачами.

1. Создайте файл модуля favorites.py

# favorites.py
import json
import os
from console_ui import console

class FavoriteManager:
    def __init__(self, connection, config):
        self.connection = connection
        self.favorites_file = config.get_config('favorites', 'file', 'favorites.json')
        self.favorites = self.load_favorites()
    
    def load_favorites(self):
        if os.path.exists(self.favorites_file):
            with open(self.favorites_file, 'r') as f:
                return json.load(f)
        return []
    
    def save_favorites(self):
        with open(self.favorites_file, 'w') as f:
            json.dump(self.favorites, f)
    
    def add_favorite(self, share_id):
        if share_id not in self.favorites:
            self.favorites.append(share_id)
            self.save_favorites()
            console.print_system_message(f'Раздача {share_id} добавлена в избранное', 'success')
    
    def remove_favorite(self, share_id):
        if share_id in self.favorites:
            self.favorites.remove(share_id)
            self.save_favorites()
            console.print_system_message(f'Раздача {share_id} удалена из избранного', 'info')
    
    def list_favorites(self):
        return self.favorites

2. Интегрируйте в client.py

from favorites import FavoriteManager

class FileHubClient:
    def __init__(self, config_path='client.conf'):
        ...
        self.favorites = FavoriteManager(self.connection, self.config)
    
    def add_favorite(self, share_id):
        self.favorites.add_favorite(share_id)
    
    def list_favorites(self):
        return self.favorites.list_favorites()

Принципы расширения:

  • ✅ Новый модуль получает только необходимые зависимости (connection, config).
  • ✅ Для постоянного хранения данных используйте JSON-файлы в директории клиента.
  • ✅ Все сообщения пользователю выводятся через console.print_system_message().
  • ✅ Для фоновых задач используйте потоки с атрибутом daemon=True.
  • ✅ Колбэки позволяют модулям реагировать на асинхронные события сервера.
