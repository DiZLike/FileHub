Документация FileHub Server

📌 Обзор проекта

FileHub — это многопользовательский сервер для обмена файлами (раздачами) в реальном времени. Клиенты подключаются по TCP, аутентифицируются, создают «раздачи» (файлы или папки) и скачивают их друг у друга. Сервер выступает координатором, проксируя данные между отправителем и получателем через отдельный канал передачи данных.

Ключевые возможности:

  • 🔐 Аутентификация с паролем или автоматическая регистрация
  • 📂 Раздачи файлов и папок с метаданными
  • 🔄 Проксирование P2P-передачи через сервер
  • 📋 Список всех активных раздач с владельцами
  • 🧹 Автоматическая очистка неактивных раздач
  • 📊 Статистика сервера и логирование
  • ⚙️ Гибкая конфигурация через .conf файл

🏗️ Архитектура

Сервер построен по модульному принципу. Каждый компонент отвечает за свою зону ответственности и взаимодействует через внедрение зависимостей.

🧠 server.py

Главный цикл, диспетчер команд, инициализация компонентов.

🔑 auth.py

Аутентификация, управление активными сессиями.

📤 shares.py

Создание, удаление, список раздач, обновление статуса.

🌐 networking.py

JSON-протокол, прокси трансферов, канал данных.

💾 storage.py

Загрузка/сохранение users.json и shares.json.

🔒 security.py

IP-фильтры, хеширование паролей, расширения.

⚙️ config.py

Парсинг filehub.conf, свойства конфигурации.

📝 logger.py

Логирование, ротация, статистика.

📡 protocols.py

Константы действий, статусов, типов.

🛠️ utils.py

Хелперы: форматирование, валидация, ID.

Все зависимости передаются в конструкторы, что упрощает тестирование и замену компонентов.

🚀 Быстрый старт

# Запуск сервера
python server.py

Сервер загрузит конфигурацию из filehub.conf (создаст автоматически при отсутствии). По умолчанию слушает порты 5000 (управление) и 5001 (данные).

Для подключения клиент отправляет JSON-сообщение с действием login.

📦 Детальное описание модулей

server.py Главный модуль

Класс FileHubServer управляет жизненным циклом: создаёт сокеты, принимает подключения, запускает фоновые потоки (очистка, сохранение, статистика). Метод _handle_client разбирает команды и вызывает соответствующие обработчики.

auth.py Аутентификация

AuthManager проверяет учетные данные, автоматически регистрирует новых пользователей (если require_password=true), ведёт словарь active_users с сокетами. Методы add_active_user / remove_active_user контролируют онлайн-статус.

shares.py Управление раздачами

ShareManager создаёт записи в storage.shares, проверяет лимиты, блокированные расширения, обновляет last_seen при пингах. Метод get_all_shares возвращает список с флагом owner_online.

networking.py Сетевое взаимодействие

NetworkManager отправляет/принимает JSON с префиксом длины (4 байта). Для передачи данных создаёт трансфер: ожидает подключения отправителя и получателя на data_port, затем проксирует байты.

storage.py Хранилище

Загружает users_data.json и shares_data.json. При старте валидирует раздачи (удаляет просроченные). Сохранение вызывается периодически и при завершении.

security.py Безопасность

Фильтрация IP (allowed/blocked), проверка расширений, хеширование паролей через PBKDF2-SHA256 с солью.

config.py Конфигурация

Читает INI-подобный filehub.conf. Предоставляет свойства server_config, security_config и др.

logger.py Логирование

Поддержка уровней, ротация по размеру, сбор статистики.

protocols.py Протоколы

Классы MessageActions, TransferRoles, StatusCodes, ShareTypes и дефолтный конфиг.

utils.py Утилиты

Статические методы: format_bytes, generate_unique_id, validate_username.

🧩 Расширение: добавление новых модулей

Архитектура FileHub позволяет легко интегрировать собственные модули. Ниже приведена пошаговая инструкция.

1. Создайте файл модуля

Например, announcements.py для системных объявлений.

# announcements.py
class AnnouncementManager:
    def __init__(self, storage, logger):
        self.storage = storage
        self.logger = logger
        self.messages = []

    def add_announcement(self, text):
        self.messages.append(text)
        self.logger.log('INFO', f'Объявление: {text}')

    def get_announcements(self):
        return self.messages[-5:]  # последние 5

2. Интегрируйте в FileHubServer

В server.py импортируйте модуль и добавьте его в __init__.

from announcements import AnnouncementManager
...
class FileHubServer:
    def __init__(self, config_path='filehub.conf'):
        ...
        self.announcements = AnnouncementManager(self.storage, self.logger)

3. Добавьте новое действие в протокол

В protocols.py расширьте MessageActions:

class MessageActions:
    ...
    ANNOUNCE = 'announce'      # новое действие
    GET_ANNOUNCEMENTS = 'get_announcements'

4. Обработайте команду в _handle_client

В методе _handle_client добавьте ветку:

elif action == MessageActions.ANNOUNCE:
    text = request.get('text', '')
    self.announcements.add_announcement(text)
    self.network.send_json(client_socket, {'status': 'ok'})
elif action == MessageActions.GET_ANNOUNCEMENTS:
    msgs = self.announcements.get_announcements()
    self.network.send_json(client_socket, {'status': 'ok', 'announcements': msgs})

5. (Опционально) Добавьте конфигурацию

В filehub.conf можно создать новую секцию [announcements] и читать её через Config.

Принципы расширения:

  • ✅ Новый класс получает только нужные зависимости (storage, logger, config).
  • ✅ Действия добавляются в MessageActions без изменения ядра протокола.
  • ✅ Обработчики в _handle_client следуют паттерну «действие → метод».
  • ✅ Для сложной логики можно создать отдельный поток (как _accept_data_connections).
